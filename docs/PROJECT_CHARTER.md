# Project Charter

## Goal

Сделать web-first сервис, который позволяет загрузить большой набор фотографий, автоматически сгруппировать их по людям и просматривать результаты через браузер.

## Primary User Story

Как пользователь фотоотчета, я хочу загрузить фото в веб-приложение, дождаться завершения фоновой обработки и открыть страницу с найденными людьми, чтобы быстро отфильтровать нужные снимки.

## Success Criteria

- Пользователь может загрузить пакет фото в веб-интерфейсе.
- Для пакета создается асинхронный job со статусами и прогрессом.
- После завершения обработки пользователь видит список person clusters, превью и связанные фото.
- Один и тот же снимок может быть связан с несколькими кластерами.
- Данные изолированы между workspace через auth и RLS.
- Vercel обслуживает только web/app layer, а тяжелая обработка выполняется отдельным worker.

## Non-Goals For Current Phase

- ручное merge/split кластеров
- продвинутая модерация качества через UI
- внешний ML API
- mobile app
- real-time collaborative editing

## Constraints

- База и storage должны находиться в Supabase.
- Web app должен деплоиться на Vercel.
- Тяжелый ML runtime нельзя считать задачей Vercel.
- Изменения публичных интерфейсов и схемы данных должны идти через спецификацию и migrations.

## Expected Inputs

- изображения JPEG/PNG
- аутентифицированный пользователь и workspace
- действие запуска обработки для загруженного батча

## Expected Outputs

- записи о job, photos, faces и clusters в Postgres
- файлы в buckets `raw-photos`, `face-previews`, `derived-artifacts`
- UI для просмотра людей, статуса job и галереи фотографий

## Operational Principles

- приватность и изоляция данных по workspace обязательны
- job lifecycle должен быть прозрачен и диагностируем
- schema changes и storage contracts должны быть воспроизводимы
- rollback для web app, worker и миграций должен быть понятен заранее
